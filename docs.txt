>   - APPEALED → Purple
>   - VOIDED / EXPIRED → Gray
>   - HIGH risk flag → Red

> - Empty states: centered illustration + heading + CTA button
> - Skeleton loaders on all data-fetching states
> - HIPAA compliance notice footer on all pages handling PHI

## Page 3 — Provider — Submit New PA Request

**[ROLE]** Provider

**[GOAL]** Submit a new Prior Authorization request with patient data, clinical codes, and supporting documents.

**[WEB LAYOUT]**
- Same sidebar + header
- Centered multi-section form card (max-width 800px)
- Breadcrumb: Dashboard > Submit PA Request
- 3-step stepper: "Patient & Payer Info" → "Clinical Codes" → "Documents & Submit"

**[COMPONENTS]**
- Step 1 — Patient & Payer Info:
  - Patient Member ID (text, 8–20 chars, alphanumeric)
  - Payer / Insurance Company (searchable dropdown, from payer_master)
  - Insurance Plan ID (dropdown, populated based on Payer selection)
  - Requesting Provider NPI (text, 10-digit, validated against NPPES registry)
  - Date of Service / Requested Date (date picker, max 90 days prior)
- Step 2 — Clinical Codes:
  - Diagnosis Codes / ICD-10 (multi-tag input: type to search ICD-10 codes, add multiple, at least 1 required)
  - Procedure / Treatment Codes (multi-tag input: CPT/HCPCS codes, at least 1 required)
  - Medication Name & Dosage (text, conditional: shown only if a Pharmacy CPT/HCPCS is selected)
  - Prior Treatment History (textarea or file upload, conditional: shown if step therapy may apply)
- Step 3 — Documents & Submit:
  - Dynamic document checklist (generated by PSA logic based on Step 1+2 inputs):
    - Each required document listed with upload zone
    - Indicator: Required / Optional tag per document
    - File format: PDF/JPG/PNG/TIFF, max 10 MB per file
  - Clinical Notes / Referral Letter (drag-and-drop upload, optional, max 10 MB)
  - Review Summary card before final submit
  - "Submit PA Request" primary button + "Back" ghost

**[DATA DISPLAYED]**
- Dynamic document checklist generated based on payer + treatment type
- Uploaded file names and sizes

**[USER ACTIONS]**
- Fill all required fields across 3 steps
- Upload all required documents
- Submit → PA created → redirect to PA detail page with confirmation

**[PAGE STATES]**
- Step-by-step navigation state
- Inline validation errors per field
- Submitting state: spinner on submit button + "Submitting to AI Pipeline…" label
- Success: toast "PA Request submitted successfully. PA ID: PA-2026-XXXXXX", redirect to detail

**[VISUAL DESIGN DIRECTION]**
- Dynamic checklist: each document row as a mini upload card — required docs have red asterisk, optional have "Optional" gray tag
- Upload zones: dashed border, upload icon, changes color on file drop
- Step indicators: accent active, completed steps with checkmarks
- Review summary in Step 3: clean read-only data card
